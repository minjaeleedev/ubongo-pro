# Ubongo 3D - Game Rules and Requirements

## 1. Game Overview

**Ubongo 3D**는 실시간 공간 추론 퍼즐 게임입니다. 플레이어는 주어진 3D 폴리오미노 조각들을 사용하여 지정된 영역을 정확히 2층 높이로 채워야 합니다. 빠른 사고력과 공간 지각 능력을 요구하며, 제한 시간 내에 퍼즐을 완성해야 합니다.

### 1.1 게임 목표
- 주어진 3D 블록 조각들을 사용하여 퍼즐 영역을 정확히 채우기
- 제한 시간 내에 퍼즐 완성
- 빠른 완성으로 보석 획득 및 점수 극대화
- 9라운드 후 가장 높은 점수 달성

### 1.2 핵심 특징
- 실시간 공간 추론 퍼즐
- 3D 폴리오미노 조각 조작
- 멀티플레이어 경쟁 지원
- 4단계 난이도 시스템
- 보석 기반 점수 시스템

## 2. Core Game Rules (핵심 게임 규칙)

### 2.1 기본 규칙
1. **퍼즐 영역**: 각 퍼즐 카드는 특정 형태의 빈 영역을 정의합니다.
2. **2층 구조**: 퍼즐 영역은 반드시 **정확히 2층 높이**로 채워져야 합니다.
3. **영역 준수**: 블록은 지정된 퍼즐 영역을 벗어날 수 없습니다.
4. **완전 충진**: 빈 공간 없이 모든 셀이 채워져야 합니다.
5. **조각 사용**: 주사위로 결정된 조각들만 사용해야 합니다.

### 2.2 조각 규칙
- 플레이어당 **8개의 3D 폴리오미노 조각** 보유
- 조각은 **회전** 가능 (X, Y, Z 축 - 총 24가지 방향)
- 조각은 **뒤집기(flip)** 가능 (원본 Ubongo 3D 규칙에 따름)
- 각 조각은 고유한 형태와 색상 보유
- 모든 조각은 3-5개의 단위 큐브로 구성 (2층 높이에 맞게 설계)

### 2.3 배치 규칙
- 조각은 그리드에 정렬되어 배치되어야 함
- 조각끼리 겹칠 수 없음
- 배치된 조각은 다시 집어서 재배치 가능
- 조각이 공중에 떠있을 수 없음 (중력 적용)

## 3. Game Modes (게임 모드)

### 3.1 Single Player Mode (싱글 플레이어)

#### 3.1.1 클래식 모드
- 개인 연습 및 기록 도전
- 9라운드 진행
- 자기 최고 기록 갱신 목표
- 난이도 선택 가능

#### 3.1.2 타임 어택 모드
- 연속 퍼즐 해결
- 총 시간 기록
- 실패 시 패널티 시간 추가

#### 3.1.3 젠 모드
- 시간 제한 없음
- 연습 및 학습 목적
- 힌트 시스템 활성화 가능

### 3.2 Multiplayer Mode (멀티플레이어)

#### 3.2.1 로컬 멀티플레이어
- 2-4명 동시 플레이
- 분할 화면 또는 순차 플레이
- 동일 퍼즐 경쟁

#### 3.2.2 온라인 멀티플레이어
- 매칭 시스템
- 랭킹 시스템
- 친구 초대 기능

#### 3.2.3 멀티플레이어 규칙
| 순위 | 보석 보상 |
|------|----------|
| 1등 | 파란 보석(3점) + 랜덤 보석 |
| 2등 | 호박 보석(1점) + 랜덤 보석 |
| 3등 | 랜덤 보석만 |
| 4등 | 랜덤 보석만 |

## 4. Difficulty System (난이도 시스템)

### 4.1 난이도 레벨

| 레벨 | 색상 | 조각 수 | 퍼즐 영역 크기 | 시간 제한 | 특징 |
|------|------|---------|---------------|----------|------|
| 1단계 | 초록 | 3개 | 작음 (3x3) | 90초 | 입문자용, 단순한 형태 |
| 2단계 | 노랑 | 4개 | 중간 (4x3) | 75초 | 기본 난이도 |
| 3단계 | 파랑 | 5개 | 큼 (4x4) | 60초 | 숙련자용 |
| 4단계 | 빨강 | 6개 | 큼+ (5x4) | 45초 | 전문가용, 복잡한 형태 |

### 4.2 난이도별 퍼즐 특성
- **1단계**: 3-4개의 유효 해결책 존재
- **2단계**: 3개의 유효 해결책 존재
- **3단계**: 2개의 유효 해결책 존재
- **4단계**: 1-2개의 유효 해결책 존재 (정밀도 요구)

### 4.3 적응형 난이도 (선택적)
- 연속 성공 시 난이도 상승
- 연속 실패 시 난이도 하락
- 플레이어 숙련도 추적

## 5. Puzzle Validation Rules (퍼즐 검증 규칙)

### 5.1 핵심 검증 조건

퍼즐이 유효하게 완성되려면 다음 조건을 **모두** 충족해야 합니다:

```
[검증 조건]
1. 영역 완전 충진: 모든 타겟 셀이 블록으로 채워져야 함
2. 2층 높이 준수: 모든 셀이 정확히 2층 높이로 채워져야 함
3. 영역 경계 준수: 블록이 퍼즐 영역 외부로 돌출되지 않아야 함
4. 조각 할당 준수: 지정된 조각만 사용해야 함 (여분의 조각 사용 불가)
5. 물리적 유효성: 블록이 공중에 떠있지 않아야 함
```

### 5.2 2층 높이 검증 알고리즘

```
Algorithm: ValidateTwoLayerHeight

Input:
  - targetArea: 퍼즐 영역의 바닥면 좌표 집합
  - placedBlocks: 배치된 모든 블록의 3D 좌표

Validation Steps:
1. FOR each (x, z) in targetArea:
   a. Count blocks at (x, 0, z) - 1층
   b. Count blocks at (x, 1, z) - 2층
   c. IF layer1_count != 1 OR layer2_count != 1:
      RETURN INVALID("2층 구조 위반")

2. FOR each placedBlock:
   a. IF block.y < 0 OR block.y > 1:
      RETURN INVALID("높이 범위 초과")
   b. IF (block.x, block.z) NOT IN targetArea:
      RETURN INVALID("영역 경계 위반")

3. RETURN VALID
```

### 5.3 검증 시점
- **실시간 검증**: 조각 배치 시 유효성 미리보기
- **완료 검증**: "완료" 버튼 클릭 또는 자동 감지
- **시각적 피드백**: 유효/무효 배치 시 색상 변화

### 5.4 에러 상태 및 피드백

| 에러 유형 | 시각적 피드백 | 메시지 |
|----------|--------------|--------|
| 영역 초과 | 빨간색 하이라이트 | "블록이 영역을 벗어났습니다" |
| 높이 초과 | 주황색 하이라이트 | "2층을 초과했습니다" |
| 빈 공간 | 노란색 깜빡임 | "빈 공간이 있습니다" |
| 겹침 | 빨간색 깜빡임 | "블록이 겹쳤습니다" |

## 6. Scoring System (점수 시스템)

### 6.1 보석 시스템 (Gem System)

원본 Ubongo 3D의 보석 기반 점수 체계:

| 보석 | 색상 | 포인트 값 |
|------|------|----------|
| 루비 | 빨강 | 4점 |
| 사파이어 | 파랑 | 3점 |
| 에메랄드 | 초록 | 2점 |
| 앰버 | 호박색 | 1점 |

### 6.2 보석 획득 규칙

#### 멀티플레이어
| 완료 순위 | 고정 보석 | 추가 보석 |
|----------|----------|----------|
| 1등 | 사파이어 (3점) | 랜덤 1개 |
| 2등 | 앰버 (1점) | 랜덤 1개 |
| 3등 | - | 랜덤 1개 |
| 4등 | - | 랜덤 1개 |
| 미완료 | - | - |

#### 싱글플레이어
| 완료 시간 | 보상 |
|----------|------|
| 남은 시간 > 75% | 루비 + 랜덤 1개 |
| 남은 시간 > 50% | 사파이어 + 랜덤 1개 |
| 남은 시간 > 25% | 에메랄드 + 랜덤 1개 |
| 시간 내 완료 | 앰버 |
| 시간 초과 | - |

### 6.3 점수 계산

```
총점 = Sum(각 보석의 포인트 값) + 보너스 점수

보너스 점수:
- 남은 시간 보너스: 남은_초 * 10
- 연속 클리어 보너스: 연속_수 * 50
- 난이도 배수: 난이도_레벨 * 1.5
```

### 6.4 랭킹 시스템
- 보석 총 가치로 1차 정렬
- 동점 시 보석 개수로 2차 정렬
- 최종 동점 시 총 클리어 시간으로 정렬

## 7. Round Progression Rules (라운드 진행 규칙)

### 7.1 게임 구성
- 총 **9라운드**로 구성
- 각 라운드는 1개의 퍼즐로 구성
- 라운드 간 짧은 휴식 시간 (선택적)

### 7.2 라운드 진행 플로우

```
Round Flow:
┌─────────────────────────────────────────────────────────┐
│ 1. 라운드 시작                                           │
│    └── 퍼즐 카드 공개                                    │
│    └── 주사위 굴림 (조각 결정)                           │
│    └── 타이머 시작                                       │
│                                                         │
│ 2. 퍼즐 해결 단계                                        │
│    └── 조각 선택 및 배치                                 │
│    └── 실시간 검증 피드백                                │
│    └── 조각 회전/재배치                                  │
│                                                         │
│ 3. 라운드 종료                                           │
│    └── 완료 확인 또는 시간 종료                          │
│    └── 점수 계산 및 보석 배분                            │
│    └── 결과 표시                                         │
│                                                         │
│ 4. 다음 라운드 전환                                      │
│    └── 보드 초기화                                       │
│    └── 새 퍼즐 준비                                      │
└─────────────────────────────────────────────────────────┘
```

### 7.3 타이머 규칙
- 라운드 시작 시 타이머 자동 시작
- 일시정지 가능 (싱글플레이어만)
- 시간 종료 시 해당 라운드 실패 처리
- 시간 종료 후에도 게임은 계속 (다음 라운드 진행)

### 7.4 주사위 시스템
- 라운드 시작 시 주사위 굴림
- 주사위 결과에 따라 사용할 조각 3-4개 결정
- 지정된 조각만으로 퍼즐 해결 필수

### 7.5 Second Chance Round (재도전 라운드)

**원본 Ubongo 3D 규칙**: 모든 플레이어가 시간 내에 퍼즐을 완성하지 못할 경우, 재도전 라운드가 시작됩니다.

#### 7.5.1 발동 조건
- **모든** 플레이어가 제한 시간 내에 퍼즐을 완성하지 못함
- 싱글플레이어 모드에서는 적용되지 않음 (시간 초과 = 라운드 실패)

#### 7.5.2 재도전 라운드 규칙
1. 타이머가 **초기화**되어 다시 시작
2. 동일한 퍼즐과 조각으로 다시 도전
3. **첫 번째로 완성한 플레이어만** 보상 획득
4. 고정 보석(사파이어/앰버) 없이 **랜덤 보석 1개만** 지급
5. 나머지 플레이어는 보석 없음

#### 7.5.3 구현 요구사항
```
Second Chance Flow:
┌─────────────────────────────────────────────────────────┐
│ 1. 시간 종료 감지                                        │
│    └── 모든 플레이어 미완료 확인                         │
│                                                         │
│ 2. 재도전 라운드 시작                                    │
│    └── "Second Chance!" 알림 표시                       │
│    └── 타이머 리셋 (동일 시간)                          │
│    └── 보드 상태 유지 (플레이어별 진행 상황 유지)        │
│                                                         │
│ 3. 재도전 라운드 종료                                    │
│    └── 첫 완성자: 랜덤 보석 1개                         │
│    └── 미완성자: 보석 없음                              │
│    └── 또다시 전원 실패: 보석 없이 다음 라운드 진행      │
└─────────────────────────────────────────────────────────┘
```

#### 7.5.4 상태 정의
| 상태 | 설명 | 다음 상태 |
|------|------|----------|
| `NormalRound` | 일반 라운드 진행 중 | `PuzzleSolved`, `TimeUp` |
| `TimeUp` | 시간 종료 | `SecondChance` (전원 실패 시), `RoundComplete` |
| `SecondChance` | 재도전 라운드 | `PuzzleSolved`, `SecondChanceTimeUp` |
| `SecondChanceTimeUp` | 재도전도 실패 | `RoundComplete` |

## 8. Victory Conditions (승리 조건)

### 8.1 싱글플레이어 승리 조건
- **게임 완료**: 9라운드 모두 플레이
- **점수 기록**: 최종 점수 및 보석 컬렉션 기록
- **등급 부여**: 점수에 따른 등급 (S, A, B, C, D)

### 8.2 멀티플레이어 승리 조건
- **9라운드 종료 후** 가장 높은 보석 가치 보유자 승리
- **동점 처리**:
  1. 보석 총 개수 비교
  2. 동일 시 플레이오프(타이브레이커) 라운드 진행
  3. 플레이오프도 동점 시 공동 우승

### 8.2.1 Tiebreaker Round (타이브레이커 라운드)

**원본 Ubongo 3D 규칙**: "If several players are tied with the highest total, a final tiebreaker round is played by only the tied players. The timer is not used, and the first player to solve the puzzle is declared the winner."

#### 타이브레이커 발동 조건
- 9라운드 종료 후 **2명 이상**의 플레이어가 **동일한 최고 보석 가치** 보유
- 보석 총 개수도 동일할 경우에만 발동

#### 타이브레이커 라운드 규칙
1. **참가자**: 동점 플레이어만 참가 (다른 플레이어는 관전)
2. **난이도**: Hard (파랑) 난이도 퍼즐 생성
3. **시간 제한**: **없음** - 순수 속도 경쟁
4. **승리 조건**: 가장 먼저 퍼즐을 완성한 플레이어 승리
5. **무승부 불가**: 타이브레이커에서 동시 완성 시 추가 라운드 진행

#### 타이브레이커 진행 플로우
```
Tiebreaker Flow:
┌─────────────────────────────────────────────────────────┐
│ 1. 동점 감지                                             │
│    └── 9라운드 종료 후 보석 가치 비교                    │
│    └── 동점자 리스트 생성                               │
│                                                         │
│ 2. 타이브레이커 준비                                     │
│    └── "TIEBREAKER!" 알림 표시                          │
│    └── 동점자만 활성화 (비동점자 UI 비활성화)            │
│    └── Hard 난이도 퍼즐 생성                            │
│    └── 타이머 UI 숨김 또는 "∞" 표시                     │
│                                                         │
│ 3. 타이브레이커 진행                                     │
│    └── 시간 제한 없이 퍼즐 해결                          │
│    └── 실시간 진행 상황 표시 (선택적)                    │
│                                                         │
│ 4. 승자 결정                                             │
│    └── 첫 완성자 = 최종 승자                            │
│    └── 결과 화면 표시                                   │
│    └── 동시 완성 시: 추가 타이브레이커 라운드            │
└─────────────────────────────────────────────────────────┘
```

#### 구현 요구사항
